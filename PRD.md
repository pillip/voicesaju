## 1. Background

### 1.1 문제 정의
- 한국 점술 시장은 **약 1조 4,000억원** 규모 (디지털/앱 1,500~2,800억원 추정).
- 19-39세 **74%가 운세를 소비**하고, 1인당 평균 **8만 300원/년** 지출.
- 그러나 기존 디지털 사주 서비스(점신·포스텔러·헬로우봇 등)는 **일방향 텍스트 + 정통/꼰대 톤**으로 20-30대 취향과 어긋남.
- "대화처럼 풀어주는 + 매운맛 + 음성" 조합은 시장에 존재하지 않음.

### 1.2 시장 기회
- MZ **럭키맥싱** 트렌드 (+39% YoY) — 사주가 "MBTI 같은 자기탐색 도구"로 소비.
- **AI 챗봇 시장 +41%** (NIA 2025), 음성 챗봇 옴니채널화 가속.
- 토스 미니앱 3,000만 유저 즉시 접근 가능.

### 1.3 우리의 차별점
1. **대화형** — 사주 풀이 + 카테고리별 꼬리질문 3개 추천
2. **매운맛 톤** — 시니컬한 누님 캐릭터 (사주), 신비로운 노인 도사 (데일리 타로)
3. **음성 출력** — Supertone TTS 캐릭터 보이스
4. **데일리 타로** — 매일 다시 오게 만드는 retention loop

## 2. Goals

### 2.1 비전
> "20-30대가 매일 한 번씩 찾아오게 만드는 한국 1위 음성 사주·타로 서비스"

### 2.2 정량 목표

| 기간 | 핵심 마일스톤 |
|------|-------------|
| **3개월** (출시 + 검증) | 가입 5,000명 · 결제 200건 · 재방문률 30% · **톤 검증 인터뷰 합격선 통과** |
| **6개월** | 가입 20,000명 · 결제 전환율 7% · 구독자 100명 · DAU/MAU 0.10 |
| **1년** | 가입 **50,000명** · 결제 전환율 8% · 구독자 500명 · **DAU/MAU 0.15** · 매출 **5~15억원** |

### 2.3 핵심 가설
- H1: "음성 + 매운맛 + 대화형"이 기존 사주 앱 대비 차별점으로 인식된다 (→ 톤 검증 인터뷰로 검증)
- H2: 데일리 타로 주 1회 무료가 retention loop를 형성한다 (→ DAU/MAU 측정)
- H3: 단건 결제 2회 = 구독 1회 가격으로 구독 전환을 자연 유도한다 (→ 결제 funnel 분석)

## 3. Target User

### 3.1 메인 페르소나

#### 🎯 P1. 직장인 효주 (28세, 여) — 메인 타겟
- IT/마케팅 사무직, 서울, 연봉 4~5천만원
- 주 1~2회 포스텔러/헬로우봇, 신년·고민 시 추가 결제
- 페인: *"포스텔러 톤은 정통, 헬로우봇은 페이월 빡세"*
- 결제 의향: 단건 5~7천원, 구독 1.5만원까지
- 매운맛 친화도: ⭐⭐⭐

#### 🎯 P2. 럭키맥싱 미주 (24세, 여) — 바이럴 엔진
- 대학생/취준생, 인스타 사주 계정 다수 팔로우
- 페인: *"사주 입력이 귀찮음, 짧게 보고 싶음"*
- 결제 의향: 단건 3.9~5.9천원 충동적
- 매운맛 친화도: ⭐⭐⭐⭐⭐

#### 🎯 P3. 워킹맘 지원 (33세, 여) — 객단가 견인
- 마케팅/영업/스타트업
- 가끔 본격 사주, "이직 운" 같은 결정 도구
- 결제 의향: 단건 1만원, 구독 미온적
- 매운맛 친화도: ⭐⭐

### 3.2 사용 채널 우선순위
1. **웹** (v1) — SEO·바이럴
2. **토스 인앱** (v1) — 3,000만 유저 즉시 접근
3. **모바일 네이티브 앱** (v2+, 우선순위 낮음)

## 4. User Stories

### 사주 풀이
- **U1**. P1로서, 카테고리(연애/직장/금전)를 선택하면 시니컬 누님이 음성으로 풀이해줘서, 점심시간에 부담 없이 운세를 듣고 싶다.
- **U2**. P1로서, 풀이 후 *"올해 이직이 좋아?"* 같은 꼬리질문을 한 번에 누를 수 있어서, 추가 입력 없이 깊게 들어가고 싶다.
- **U3**. P3로서, 내 사주 명식(천간·지지)을 시각화해서 보고, 풀이의 근거를 이해하고 싶다.

### 데일리 타로
- **U4**. P2로서, 매일 카드 1장을 뽑아 30~40초 음성 풀이를 듣고, 짧은 하루의 위로를 받고 싶다.
- **U5**. P2로서, 풀이가 끝나면 자동 생성된 명대사 카드를 인스타 스토리로 공유하고 싶다.

### 결제·구독
- **U6**. P1로서, 단건 4,900~7,900원이면 충동 결제하고, 두 번째 결제 시 구독이 더 이득임을 알 수 있어서, 자연스럽게 구독으로 전환하고 싶다.
- **U7**. P1로서, 토스 인앱에서는 토스페이 원클릭으로 결제하고, 웹에서는 카카오페이/토스페이를 선택할 수 있어야 한다.

### 마이페이지 / 재방문
- **U8**. P3로서, 가입 후 사주 정보를 영구 저장하고, 과거 풀이를 다시 들을 수 있어야 한다.
- **U9**. P1로서, 사주 정보를 잘못 입력했을 때 2회까지 무료로 수정할 수 있어야 한다.

### 비회원 체험
- **U10**. 비회원으로서, 회원가입 없이 첫 사주 풀이 1회를 체험하고, 마음에 들면 1초 가입으로 결과를 저장하고 싶다.

## 5. Functional Requirements

### 5.1 온보딩 입력 (단계별 카드)

```
Step 1: 생년월일 (양력/음력 토글)
Step 2: 태어난 시각 (시간·분 선택 + "모름" 체크박스)
Step 3: 성별 (여/남)
Step 4: 이름 (옵셔널 — 입력 시 캐릭터 호칭에 반영)
→ 카테고리 선택 화면으로 진입
```

- "시각 모름" 모드: 시주(時柱) 비우고 일간·월지 위주 풀이 + 안내 멘트 *"시간을 모르면 큰 줄기는 보지만 디테일은 흐릿해"*.
- 비회원도 가능. 가입은 풀이 후 *"결과 저장하려면 1초 가입"* 시점에 유도.

### 5.2 사주 풀이 흐름

```
① 카테고리 선택 (연애 / 직장 / 금전)
   ↓
② 캐릭터 인트로 15초 (미리 녹음된 정형 멘트 — 매운맛 한 줄 맛보기)
   "어디 한번 봅시다… 28살 무자년… 음, 재미있네."
   ↓
③ 페이월 — "본격 풀이 받으려면 결제"
   - 단건 결제 (4,900~7,900원) OR 구독 가입 OR 신규 가입 1회 무료 토큰 사용
   ↓
④ 결제 완료 → LLM 호출 (Claude Sonnet 4.6) → 풀이 생성 → Supertone TTS
   ↓
⑤ 본 풀이 재생 (1~2분, 자동 재생, 자막 동시 표시)
   - 캐릭터 일러스트 + 자막 흐름 + 사주 명식 한 줄 요약
   - 컨트롤: 일시정지 / 다시 듣기 (구간 이동은 v2)
   ↓
⑥ 풀이 종료 → 꼬리질문 추천 버튼 3개 자동 노출
   예) "이 사람이랑 결혼해도 돼?" / "올해 이직이 좋아?" / "빚 갚을 수 있어?"
   ↓
⑦ 꼬리질문 클릭 → Claude Haiku 4.5 호출 → 30~40초 답변 음성 (Supertone)
   - 본 풀이 결제에 꼬리질문 3개 포함 (별도 결제 없음)
   - 클릭된 버튼은 비활성화, 나머지 2개는 계속 클릭 가능
   - 3개 모두 클릭 또는 "이만 마칠게요" 버튼 → 세션 종료
   - v1은 답변 후 추가 꼬리질문 추천 없음 (단순화)
   ↓
⑧ 명대사 카드 자동 생성 + "공유하기" 버튼
```

### 5.3 데일리 타로 흐름

- **카드**: 22장 메이저 아르카나 (자체 일러스트 — 노인 도사 세계관과 일관)
- **결정성 시드**: `SHA256(date_KST + user_id)[:8] mod 22` → 같은 날 새로고침 시 동일 카드
- **인터랙션**: 카드 뒷면 → 탭 → 뒤집힘 애니메이션 → 노인 도사 음성 자동 재생 (30~40초)
- **무료 한도**: 화면 상단 *"이번 주 무료 1회 남음"* 배너 → 소진 시 결제·구독 페이월
- **종료 후**: 명대사 카드 자동 생성

### 5.4 명대사 카드 (바이럴)

- **트리거**: 사주 풀이 + 데일리 타로 종료 직후 자동 생성
- **구성**: ① 매운맛 명대사 1줄 (LLM이 풀이 중 가장 임팩트 있는 한 줄 추출) ② 캐릭터 일러스트 ③ 카테고리/카드명 ④ `VoiceSaju` 워터마크
- **디자인 변형 (카테고리별 색상)**:
  - 연애 = 핑크 / 직장 = 블루 / 금전 = 골드 / 타로 = 퍼플
- **생성 방식**: **서버 OG 이미지 생성** (1080×1920 인스타 스토리 사이즈) — 공유 링크 미리보기로 SEO·바이럴 부스트
- **공유 채널**: 인스타 스토리 (1080×1920) / 카톡 / 이미지 다운로드

### 5.5 결제·구독

| 티어 | 내용 | 잠정 가격 |
|------|------|----------|
| **무료** | 데일리 타로 주 1회 + 신규 가입 시 사주 1회 무료 토큰 | — |
| **단건** | 사주 1회 결제 (본 풀이 1~2분 + **꼬리질문 3개 × 30~40초 포함**) | 4,900~7,900원 |
| **월 구독** | 사주 월 1회 + 데일리 타로 매일 1회 | 9,900~14,900원/월 |

- **결제 수단**:
  - 토스 인앱 → **토스페이 원클릭**
  - 웹 → **토스페이 + 카카오페이**
- **단건 → 구독 자연 유도**: 단건 2회 ≈ 구독 1회 → 단건 결제 후 *"매달 사주 + 매일 타로가 X원에 다 됩니다"* 페이지 노출
- **자동 환불 정책**: LLM/TTS 실패 시 자동 환불 또는 무료 풀이 1회 적립

### 5.6 회원 / 비회원 분기

| | 비회원 | 회원 |
|---|---|---|
| 첫 풀이 1회 무료 | ✅ | ✅ (가입 시 토큰 1개 지급) |
| 데일리 타로 주 1회 무료 | ✅ (디바이스 ID 기준) | ✅ |
| 사주 데이터 영구 저장 | ❌ | ✅ |
| 결제/구독 | ❌ | ✅ |
| 풀이 히스토리 다시 듣기 | ❌ | ✅ |
| 기기 간 동기화 | ❌ | ✅ |

- **회원가입 수단**: 토스 인앱 = 토스 ID 자동 / 웹 = **카카오 + 애플 소셜 로그인**
- **회원 전환 시점**: 비회원 첫 풀이 후 *"결과 저장하려면 1초 가입"* CTA

### 5.7 마이페이지

1. **내 사주 명식** — 천간·지지·오행 표 시각화
2. **풀이 히스토리** — 과거 풀이 음성 다시 듣기 (스트리밍, 재생성 X)
3. **결제·구독 관리** — 단건 내역, 구독 상태, 구독 해지
4. **사주 정보 수정** — 2회 무료, 이후 운영 문의 (어뷰징 방지)
5. **알림 설정** (v2 푸시 도입 시)

## 6. Non-functional Requirements

### 6.1 성능

| 항목 | 기준 |
|------|------|
| 인트로 재생 시작 → 본 풀이 첫 음성 | **3초 이내** (LLM 스트리밍 + TTS 청크 합성) |
| Supertone 첫 청크 응답 | **1.5초 이내** |
| 데일리 타로 카드 뒤집기 → 음성 재생 | **2초 이내** |
| 꼬리질문 클릭 → 답변 첫 음성 | **2초 이내** (Haiku 4.5 + Supertone) |
| 풀이 1회당 LLM+TTS 비용 상한 | **단건가의 20% 이하** (본 풀이 + 꼬리질문 3개 합산) |

### 6.2 보안 / 개인정보

- 생년월일·시각 = 민감정보 → **DB 컬럼 단위 암호화** (AES-256)
- 개인정보처리방침 명시, 통신판매업 신고 완료
- 약관에 **"오락 목적" 면책** 명시 (업계 표준)
- 결제 정보는 PG사(토스페이먼츠) 위임 — 자체 저장 X

### 6.3 접근성

- WCAG 2.1 AA 색상 대비
- 자막 동시 표시 (이미 디자인 결정)
- 키보드 네비게이션 (웹)
- 모바일 우선 디자인 (반응형)

### 6.4 안정성 / 오류 처리

| 시나리오 | 대응 |
|---------|------|
| 결제 실패 | 풀이 시작 전 차단, 재시도 UI |
| LLM 실패 | 캐릭터 캐싱 멘트 *"별기운이 잠시 약하네…"* + 자동 환불/토큰 적립 |
| TTS 실패 | 텍스트만 표시 fallback (자막 이미 존재) |
| 네트워크 끊김 | 음성 일시정지 → 재연결 시 이어듣기 |
| 톤 가드레일 위반 (욕설/혐오) | 실시간 필터 → 차단 시 안전한 대체 멘트 |

### 6.5 가드레일 (LLM 톤)

- 시스템 프롬프트로 톤 강도 컨트롤 (매운맛 ≠ 욕설)
- **톤 평가셋 50~100개** (양호/위반 케이스) → 출시 전 회귀 테스트
- 실시간 필터: 욕설·혐오·성희롱·차별 표현 차단 → 안전 멘트로 대체

## 7. Out of Scope (v2+)

### v1 제외 — v2 이후 검토
- **STT (유저 음성 입력)** — 마이크 권한·환경·정확도 부담
- **모바일 네이티브 앱** (iOS/Android)
- **푸시 알림 / 카카오 알림톡**
- **캐릭터 선택** (v1은 사주=시니컬 누님, 타로=노인 도사 고정)
- **추가 카테고리** (건강 / 대인관계 / 학업 / 가족 등)
- **다중 카드 스프레드** (3장 / 켈틱 크로스 등)
- **궁합 / 연간운세 / 합방 풀이**
- **꼬리질문 자유 텍스트 입력** (v1은 추천 버튼 3개만)
- **풀이 구간 이동** (v1은 일시정지/다시 듣기만)
- **알림 / 리텐션 후킹** (v1은 자발 방문에 의존)

## 9. Technical Notes

### 9.1 기술 스택

| 영역 | 선택 |
|------|------|
| **프론트엔드** | Next.js (TypeScript, App Router) — SSR/OG 이미지 강점, 토스 WebView 그대로 호환 |
| **백엔드** | Python + FastAPI (`uv` 패키지 관리, `pytest`) — 사주 만세력 라이브러리 생태계 풍부 |
| **LLM** | **Claude Haiku 4.5** (인트로/타로) + **Claude Sonnet 4.6** (사주 본 풀이/꼬리질문) 하이브리드 |
| **TTS** | **Supertone** — 시니컬 누님 + 노인 도사 캐릭터 보이스 |
| **만세력** | 1순위 `manseryeok` (PyPI) + 음양력 변환 `korean-lunar-calendar` (PRD 진행 중 후보 비교) |
| **DB** | PostgreSQL (사용자·사주·결제·풀이 히스토리), Redis (세션·결정적 타로 시드 캐시) |
| **인증** | 카카오 + 애플 소셜 로그인 (웹), 토스 ID (인앱) |
| **결제** | 토스페이먼츠 SDK (토스페이 + 카카오페이) |
| **OG 이미지** | `@vercel/og` (Edge Runtime) 또는 Pillow 서버 생성 |
| **호스팅** | 프론트 = Vercel / 백엔드 = Fly.io 또는 Railway (PRD 진행 중 결정) |

### 9.2 LLM 톤 가드레일 정책

1. **시스템 프롬프트** — 캐릭터 톤 / 매운맛 강도 / 금지어 명시
2. **톤 평가셋** — 50~100개 케이스 (양호/위반 분류) → 출시 전 회귀 테스트
3. **실시간 필터** — 욕설·혐오·성희롱·차별 차단 (한국어 모더레이션 + 자체 deny list)
4. **출시 전 모델 비교** — Claude Sonnet 4.6 / Haiku 4.5 / GPT-4o mini 톤 샘플 5개 비교

### 9.3 사주 명식 계산 분리 원칙

- **명식 산출은 자체 엔진 (룰 기반, 결정적)** — LLM에 절대 위임하지 않음
- **LLM은 해석·매운맛 톤만 담당** — 명식(천간/지지/오행/십신)을 시스템 프롬프트로 주입
- 동일 사주는 동일 명식 결과 보장 (시드 캐싱)

### 9.4 토스 인앱 정책 사전 확인 (필수)

- 결제 SDK 제약 (토스페이 외 결제 가능 여부)
- 인증 방식 (토스 ID 외부 연동 가능 여부)
- 콘텐츠 정책 (매운맛 톤이 토스 가이드라인 통과 가능한가)
- WebView 권한 (마이크는 v1 미사용이나 향후 STT 위해 정책 확인)

### 9.5 성능·비용 시뮬레이션 (PRD 단계 실시)

- Supertone 티어 결정 — 월 풀이 횟수 가정 후 크레딧 계산 (1회당 본풀이 1~2분 + 꼬리질문 3개 × 30~40초 = **총 2.5~4분 음성**)
- Claude API 비용 — 본 풀이(Sonnet 4.6) + 꼬리질문 3개(Haiku 4.5) input/output 토큰 평균 측정
- 단건가 4,900~7,900원 × 20% = **풀이당 980~1,580원 안에서 LLM(본풀이+꼬리질문3) + TTS 합산**이 들어와야 함
- 꼬리질문에 Haiku 4.5(Sonnet 대비 ~1/5 비용) + 답변 30~40초로 비용 효율 확보
