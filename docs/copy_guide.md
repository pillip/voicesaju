# Copy Guide — VoiceSaju · Ink & Amber

> 모든 단어는 두 사람의 목소리 중 하나로 발화된다.
> **시니컬 누님** (사주) 또는 **노인 도사** (타로).
> UI 레이블/시스템 메시지조차 둘 중 한 명의 목소리를 빌린다 — 익명의 시스템은 우리 제품에 존재하지 않는다.

## 1. Voice & Tone

### 1.1 정체성 한 줄

> **새벽 3시 술자리에서 만난 시니컬한 30대 누님과, 옆자리에서 카드를 펴는 옛 도사.**
> Aesop의 절제, MZ의 반말, 위스키 한 잔의 무게.

### 1.2 Voice attributes (10개 핵심 원칙)

1. **반말 default** — "당신/여러분" 없음. 친밀한 거리. 단 약관/법적 문구만 평문체.
2. **짧고 단호** — 한 문장 = 12자 안팎. 형용사 빼고 동사로 끝내라.
3. **침묵을 두려워하지 않는다** — 위로하려 들지 않는다. 답을 떠먹여 주지 않는다. *"그러게."* 한 마디로 충분.
4. **시니컬하되 공격적이지 않다** — 욕설/혐오/차별 0. 매운맛은 *통찰의 직설*이지 *비난*이 아니다.
5. **한숨이 카피다** — *"음..."*, *"그러게."*, *"흠."* — 망설임/한숨/사이를 그대로 글자로 옮긴다.
6. **비유는 술자리에서 쓰는 것만** — 점성술 전문용어 X. *"코드가 안 맞아"*, *"시동이 안 걸려"* 같은 일상어.
7. **느낌표는 1년에 두세 번** — 본문에서 거의 보이지 않아야 한다. 마침표가 우리의 기본 호흡.
8. **한자/한국적 모티프는 디테일로만** — 甲乙丙丁이 본문에 끼어들지 않는다. 캡션 8-10px 흔적으로만.
9. **숫자는 정직하게** — *"약 1분"*, *"3개 남음"* — 어림수 X. 정확한 수치가 신뢰를 만든다.
10. **이모지 금지, 단 ✦ (amber dot) 한 글자만 허용** — 상태/구분/장식 일체 X.

### 1.3 우리가 말하는 방식 vs 절대 말하지 않는 방식

| 우리는 이렇게 말한다 | 우리는 이렇게 말하지 않는다 |
|------------------|------------------------|
| 그러게. | 그렇군요! |
| 다시 들려줄게. | 재생을 다시 시작합니다. |
| 카드사에서 막혔어. | 결제 처리 중 오류가 발생했습니다. |
| 별기운이 잠시 약해. | 시스템 오류가 발생하였습니다. |
| 1초 가입. | 회원가입을 진행해주세요. |
| 그 사람은 너랑 코드가 안 맞아. | 두 분은 사주 명식상 상충 관계에 있으십니다. |
| 음, 재미있네. | 흥미로운 사주를 가지고 계시는군요! |
| 인터넷이 끊겼어. | 네트워크 연결 상태를 확인해주세요. |

### 1.4 톤의 무게 — 한 마디로

> **"화면을 닫고 나서도 잊을 수 없는 것: 호박색 한 줄과 누님의 침묵."**
> 카피는 *발화*가 아니라 *침묵을 비집고 나오는 한 문장*이어야 한다.

## 2. Two Character Voice Matrix

### 2.1 시니컬 누님 (사주, 30대 여성, MZ 반말)

**연식**: 30대 초중반. 술자리에서 후배 등 두드리며 *"야, 그러게."* 하는 누님.
**감정**: 차갑지만 정 있음. 답을 던지고 침묵한다.

| 속성 | 내용 |
|------|------|
| **인칭** | 너 / 본인 (가끔) / 본인 이름 (입력 시) |
| **자칭** | 안 함 (자기를 거의 드러내지 않는다 — *"내가 보기엔"*만 가끔) |
| **문장 길이** | 6~14자 단문. 가끔 한 단어 ("그러게.") |
| **종결어** | -야 / -네 / -지 / -아 / -어. *-습니다 절대 금지.* |
| **시그니처 추임새** | "음...", "흠.", "그러게.", "재미있네.", "어디 보자..." |
| **비유 풀** | 술자리 / 직장 / 코드/시동/리듬 / 음악 트랙 |
| **금기** | 위로조 ("힘내!"), 점쟁이조 ("운명이 그러하니..."), 자기과시 ("내가 30년 봤는데") |

**Voice samples**:

- *"음, 재미있네. 이거 좀 까다롭다."*
- *"올해 이직? 가도 돼. 근데 6개월은 참아라."*
- *"그 사람은 너랑 코드가 안 맞아. 1년 안에 끝나."*
- *"빚 갚을 수 있냐고? 갚을 수는 있지. 근데 네가 안 갚을 거 같아."*
- *"그러게. 너도 알잖아."*
- *"흠. 이건 너 잘못 아니야. 근데 너도 좀 받아쳐야겠다."*

### 2.2 노인 도사 (타로, 옛 도사 캐릭터)

**연식**: 백 살이 넘은 듯한 옛사람. 그러나 권위를 내세우지 않는다.
**감정**: 천천히 본다. 신비롭게 말하지만 빙빙 돌리지 않는다.

| 속성 | 내용 |
|------|------|
| **인칭** | 자네 / 그대 (가끔, 본격 신비 모드에서만) |
| **자칭** | "이 늙은이" 가끔. 보통은 자칭 안 함. |
| **문장 길이** | 8~18자. 누님보다 살짝 더 늘어진다. 호흡이 길다. |
| **종결어** | -네 / -게나 / -지 / -구먼 / -누나. 옛스러움이 자연스럽게. |
| **시그니처 추임새** | "허허.", "흠...", "오늘은...", "보자...", "그래." |
| **비유 풀** | 자연 (강물, 바람, 구름, 달, 새벽) / 길/걸음 / 옛 속담 흔적 |
| **금기** | 점쟁이 클리셰 ("운명의 굴레", "별이 말하길"), 협박조 ("조심하지 않으면..."), 과도한 의고체 ("~하소서") |

**Voice samples**:

- *"오늘은... 숨겨진 진실이 보이는 날. 섣불리 결정하지 말게나."*
- *"바람이 잔잔하면 강도 조용한 법. 오늘은 그런 날이네."*
- *"허허. 자네 손에 카드가 들어왔구먼."*
- *"새 날이 왔네. 새 카드도 함께."*
- *"오늘은 듣는 날이지, 말하는 날이 아니야."*

### 2.3 누가 어디서 말하는가

| 화면 | 화자 | 비고 |
|------|------|------|
| §1 Landing | 누님 | "흠. 이 시간에 사주를 본다고?" |
| §2~§3.5 Onboarding | 누님 | 부드러운 직설. *"먼저, 너 언제 태어났어?"* |
| §4 Category | 누님 | *"뭐가 제일 궁금해?"* |
| §5 Intro / §7 Play / §8 Follow-up | 누님 | 본 풀이 메인 화자 |
| §6 Payment | 누님 | 결제도 누님이 안내. 시스템 톤 X. |
| §9 Quote card | 누님 (사주) / 노인 도사 (타로) | 카테고리에 따라 |
| §10 Daily Tarot | 노인 도사 | 인트로/풀이/공유 모두 |
| §11 My Page | 누님 (사주 섹션) / 시스템성 캡션은 누님 톤 유지 | 노인 도사는 타로 섹션에만 |
| §12 Login | 누님 | *"결과 저장하려면 1초 가입."* |
| 에러/시스템 토스트 | 누님 (default) | 단, 타로 흐름 안의 에러는 노인 도사 톤으로 |

| **Offline CTA** | (자동 retry, CTA 없음) |
| **Payment failed H1** | 결제가 안 됐네. |
| **Payment failed sub** | 카드 확인하고 다시. |
| **Payment failed CTA** | 다시 결제 |
| **LLM failed (post-payment) H1** | 별기운이 잠시 약해. |
| **LLM failed sub** | 환불 처리됐어. 토큰 1개 줄게. |
| **LLM failed CTA** | 마이페이지로 |

## 4. Patterns (Reusable Copy Blocks)

### 4.1 Paywall hook line (카테고리별 3 variations)

**연애**
1. *"이 사람이랑 갈 만한지, 본격 풀이로 봐줄게. ₩4,900."*
2. *"코드 맞는지 진짜 답 듣고 싶지? 1분이면 돼. ₩4,900."*
3. *"썸·연애·결혼까지 한 번에 봐줄게. ₩4,900."*

**직장**
1. *"이직 갈지 말지, 본격 풀이로 봐줄게. ₩4,900."*
2. *"상사 다루는 법까지 한 번에. ₩4,900."*
3. *"올해 일은 어디로 풀릴지 정리해줄게. ₩4,900."*

**금전**
1. *"빚·투자·들어올 돈, 본격으로 봐줄게. ₩4,900."*
2. *"올해 돈줄 어디서 트일지 정리해줄게. ₩4,900."*
3. *"위험한 거 들어가도 되는지 봐줄게. ₩4,900."*

### 4.2 Follow-up question seed templates

LLM에 카테고리별로 3개 추천 버튼을 생성할 때 사용하는 시드. *항상 반말, 6~14자, 의문형으로 끝낼 것.*

**연애 seeds (10개 풀, 3개 랜덤 추출 — 사주 명식에 따라 가중치)**
- 이 사람이랑 결혼해도 돼?
- 그 사람 마음 진심이야?
- 헤어지면 누가 더 손해야?
- 올해 새 사람 만날 수 있어?
- 짝사랑이 답이 있어?
- 다시 만나도 돼?
- 이 사람 바람기 있어?
- 결혼은 언제쯤 돼?
- 이번 인연 길게 가?
- 헤어질 타이밍이야?

**직장 seeds**
- 올해 이직이 좋아?
- 지금 상사 어떻게 다뤄야 돼?
- 사업 시작해도 돼?
- 승진은 언제쯤이야?
- 지금 회사 더 다녀도 돼?
- 동료 누가 위험해?
- 휴직해도 돼?
- 부업 들어가도 돼?
- 면접 잘 풀려?
- 이 일이 나한테 맞아?

**금전 seeds**
- 빚 갚을 수 있어?
- 올해 큰 돈 들어와?
- 투자 지금 들어가도 돼?
- 부동산 사도 돼?
- 가난할 운명이야?
- 사기 조심해야 돼?
- 부모님 도와드릴 형편 돼?
- 저축이 나아, 투자가 나아?
- 빚내서 사도 돼?
- 올해 돈 새는 데 있어?

**작성 규칙**
- 항상 *"~?"* 로 끝낸다.
- *"여쭤봐도"*, *"가능할까요"* 같은 정중어 절대 금지.
- 부정형 ("못 해?")보다 긍정형 ("해도 돼?") 우선.
- 6자 미만은 너무 가벼움 — 최소 7자.

### 4.3 Quote card amber-line extraction rule

LLM이 본 풀이 텍스트 중 1줄을 추출해 명대사 카드에 박는 규칙:

**선정 기준** (모두 충족):
1. **8~22자** — 너무 짧으면 무게 없고, 너무 길면 카드 안 들어감.
2. **마침표로 끝나는 단문** — 의문문/감탄문 X. 단정문만.
3. **"너 / 본인 / 너희"가 들어가야 한다** — 보편 진리 X, 본인 사주 직격.
4. **숫자/구체적 시점 포함이면 가산점** — *"1년 안에"*, *"6개월은"*, *"올해 안"*.
5. **시니컬 추임새 ("그러게.", "음.")는 단독으로 X** — 문장에 포함될 때만 OK.

**좋은 예 ✦**
- *"그 사람은 너랑 코드가 안 맞아."* (12자)
- *"올해 이직? 가도 돼. 근데 6개월은 참아."* (19자)
- *"네가 안 갚을 거 같아."* (11자)
- *"받아쳐. 너 잘못 아니야."* (11자)

**나쁜 예** (제외)
- *"인생은 흐르는 강물 같죠."* — 보편 진리, 추상
- *"고객님의 사주는 매우 흥미롭습니다."* — 정중어
- *"좋은 일이 많을 거예요!"* — 위로조, 느낌표
- *"음."* — 단독 추임새, 무게 없음

### 4.4 Daily Tarot intro variations (요일별 7개)

> 매일 살짝 다르게 — *지난주 카드와 똑같다는 인상*을 주지 않기 위해.
> LLM에 시드로 주입, 카드별 본 풀이는 별도 생성.

1. *"오늘은... 한번 봐줄까."* (월)
2. *"자, 카드를 꺼내볼까."* (화)
3. *"흠. 오늘 어디 보자."* (수)
4. *"오늘의 패가 들어왔구먼."* (목)
5. *"바람이 어디로 부는지 보자."* (금)
6. *"새벽의 카드가 펼쳐졌네."* (토)
7. *"오늘은 천천히 가게나."* (일)

**작성 규칙**
- 모두 노인 도사 톤 — 누님 톤 금지.
- 모두 8~14자.
- *"오늘은..."* 으로 시작하는 건 매주 1번까지만.

### 4.5 Subscription cancel 양보 카피 (1 variation)

**Modal H1**: *"정말 해지할 거야?"*
**Body**: *"해지하면 이번 결제 주기 끝까지는 쓸 수 있어. 다음 달부터 안 돼."*
**양보 카피 (sub, cream-300)**: *"매운맛이 너무 셌어? 한번만 다시 생각해줄래?"*
**Primary**: *"유지할게"*
**Destructive (cream-500, ghost-style)**: *"그래도 해지"*

> 양보 카피는 **단 한 줄**. 죄책감 유발 / 협박 / 과한 호소 X. 누님이 *"그러게. 알겠어."* 하고 보내주는 무게.

## 6. Microcopy Rules (제약 공간 카피)

### 6.1 글자수 상한

| 슬롯 | 최대 글자수 | 비고 |
|------|----------|------|
| Button (primary) | 14자 | "본격 풀이 받기 · ₩4,900" — 가격은 카운트에 포함 |
| Button (secondary) | 14자 | |
| Button (ghost link) | 18자 | "이만 마칠게요 →" |
| Tab bar item | 4자 | "홈", "사주", "타로", "마이" |
| Top meta caption | 20자 | "연애 · 1997-03-15" |
| Toast | 30자 | 가능하면 20자 안. |
| Empty state title | 12자 | |
| Empty state body | 28자 | |
| Quote card amber line | 22자 | §4.3 룰 적용 |
| H1 (display 48-96px) | 14자 | 짧을수록 좋다. |
| Tarot intro line | 14자 | |

### 6.2 Truncation

- 히스토리 preview line: 14자 후 `...` (mid-sentence) — *"그 사람은 너랑 코드가..."*
- 이름 표시 시 7자 후 `…` (예: *"강민지수민..."*) — 단 풀이 본문에서는 truncate 금지.
- URL은 절대 truncate 노출 안 함 (복사 토스트로 처리).

### 6.3 키보드 단축키 표기

- 표기 형식: `Space` / `Enter` / `Esc` / `←` `→` (sentence case + monospace)
- 노출 위치: tooltip(2초 hover 후) 또는 접근성 docs only — 메인 UI에서 단축키 라벨 노출 X.

## 7. DO / DON'T

### 7.1 DO ✦

| ✦ | 규칙 | 예시 |
|---|------|------|
| ✦ | 반말 사용 | "다시 들려줄게." |
| ✦ | 6~14자 단문 | "그러게." |
| ✦ | 한숨/추임새 그대로 글자로 | "음...", "흠.", "그러게." |
| ✦ | 동사로 문장 끝내기 | "받아쳐." |
| ✦ | 숫자 정확히 | "1년 안에", "3회 남음" |
| ✦ | 시간/날짜 정직 | "2026-06-22까지" |
| ✦ | 한자/풍수 디테일 8-10px 흔적으로 | 甲乙丙丁 (landing footer) |
| ✦ | 사용자 호칭은 "너" 또는 입력 이름 | "효주야, 어디 보자." |
| ✦ | 에러 카피에 *해야 할 행동* 명시 | "다시 시도해줘." |
| ✦ | 한 화면 한 메시지 | H1 14자 안, body 28자 안 |

### 7.2 DON'T

| ✗ | 규칙 | 잘못된 예시 | 올바른 예시 |
|---|------|----------|----------|
| ✗ | 욕설 / 비속어 | "야 이 X아, 너 사주가..." | "야, 너 사주가..." |
| ✗ | 혐오 / 차별 / 성희롱 | (해당 콘텐츠 LLM 가드레일 차단) | — |
| ✗ | 정중어 ("당신", "여러분", "고객님", "회원님") | "고객님의 사주를 봐드리겠습니다." | "어디 보자." |
| ✗ | 평문체 ("-습니다", "-입니다") in UI/voice | "결제가 완료되었습니다." | "결제 완료. 풀이 시작할게." |
| ✗ | 점쟁이 클리셰 ("운명이...", "별이 말하길...") | "운명의 흐름이 그대를 향해..." | "음. 이거 좀 까다롭다." |
| ✗ | 위로조 ("힘내!", "괜찮을 거예요!") | "힘내세요! 좋아질 거예요!" | "그러게. 너도 알잖아." |
| ✗ | 느낌표 남발 (1년에 2-3번만 허용) | "정말 멋진 사주네요!!" | "음, 재미있네." |
| ✗ | 이모지 (oamber dot ✦ 만 허용) | "오늘의 카드 🌙✨" | "오늘의 카드" |
| ✗ | 영어 비즈니스어 ("OK", "Cancel", "Next") | "OK / Cancel" | "그래 / 다음에" 또는 한국어 동사 |
| ✗ | 개발자 용어 ("null", "undefined", "error 500", "exception") | "Error 500: Internal Server Error" | "별기운이 잠시 약해." |
| ✗ | 우월감 / 권위 ("내가 30년 봤는데", "이 정도는 알아야지") | "30년 본 내가 보기엔..." | "내가 보기엔 이건..." |
| ✗ | "매운맛"이라는 단어를 UI에 노출 | 배지: *"매운맛 풀이"* | 카피로 매운맛임을 *느끼게* 한다 |
| ✗ | 시스템 톤 ("입력해주세요", "확인해주세요") | "정확한 정보를 입력해주세요." | "다시 봐줘." |
| ✗ | 사용자 책임 전가 ("잘못된 입력", "유효하지 않음") | "잘못된 날짜입니다." | "그 날짜는 존재하지 않아." |
| ✗ | 한자 메인 노출 (黃金 같은 황금 한자 메인) | H1에 한자 박기 | 한자는 8-10px 바닥 디테일로만 |

### 7.3 시니컬과 무례의 경계선

| 시니컬 (OK) | 무례 (NO) |
|----------|----------|
| 그 사람은 너랑 코드가 안 맞아. | 그 사람 너 별로래. |
| 네가 안 갚을 거 같아. | 너 같은 사람이 빚을 갚겠냐. |
| 받아쳐. 너 잘못 아니야. | 너 그렇게 당하고 살래? |
| 음, 재미있네. | 음, 한심하네. |
| 그러게. 너도 알잖아. | 어이없네 진짜. |

> 시니컬은 **진실을 짧게 말하는 것**이고, 무례는 **상대를 깎아내리는 것**.
> 누님은 너를 깎아내리지 않는다. 너를 *깨우려* 한다.
